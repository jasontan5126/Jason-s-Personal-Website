SPI Transmitter and Receiver (Spring 2020)


Description: Programmed in VHDL (VHSIC Hardware Description Language) - Simulated waveforms are generated in which each bit of data input from MSB is being transmitted to the SPI receiver to receive each bit of data from the SPI transmitter.


SCK is a serial clock that controls the shifting of data in the interfacing it is a clock signal.


SS_Bar (Slave Select Bar) is the control signal that signals to the slave device that data is ready to be transmitted in this case when SS_Bar goes from a logic 1 to logic 0.


POL (Clock Polarity) controls the polarity of the clock. Since SCK doesn't stays a constant value until its ready to transmit data CPOL determines that. If CPOL=1 clock stays at logic 1 when not transmitting data and the opposite when 0.


CPHA (clock phase) determines which edge data is shifted at relative to the clock pulses. If CPHA is a 1 a but is transferred at a leading edge of the clock and if CPHA is a a 0 its transferred at a trailing edge of clock.



NOTE: I was not able to synthesize it in the FPGA (Field-Programmable Gate Array) b/c of COVID. Plus the lab room was closed during that time.
